# Advanced Event-Driven User Behavior Tracking (Google Cloud + MongoDB)

## Architecture
Frontend tracker script
-> in-browser event queue
-> batch send every 5-10s (`navigator.sendBeacon`, fallback `fetch keepalive`)
-> backend tracking endpoint (`POST /api/track`)
-> async Pub/Sub publish (`user-behavior-events`)
-> analytics worker subscriber
-> MongoDB analytics collections
-> admin behavior dashboard (`/behavior-analytics`)

Main ecommerce API logic never waits for analytics DB writes.

## Mongo Collections

- `cart_events`
- `purchases`

### Core indexes
- `events_raw`: `eventId` unique, `sessionId`, `userId`, `eventType`, `timestamp`, TTL on `expiresAt`
- `sessions`: `sessionId` unique, `userId`, `startedAt`
- `page_views`: `sessionId`, `pageUrl`, `startedAt`
- `product_events/cart_events/purchases/search_events`: query indexes on entity fields + timestamp

## Pub/Sub Setup (GCP)
```bash
PROJECT_ID="your-project-id"
TOPIC="user-behavior-events"
SUB="user-behavior-events-sub"
DLQ="user-behavior-events-dead-letter"

# APIs
gcloud services enable pubsub.googleapis.com run.googleapis.com artifactregistry.googleapis.com --project "$PROJECT_ID"

# Topic + DLQ
gcloud pubsub topics create "$TOPIC" --project "$PROJECT_ID"
gcloud pubsub topics create "$DLQ" --project "$PROJECT_ID"

# Subscription with retry/dead-letter
gcloud pubsub subscriptions create "$SUB" \
  --topic="$TOPIC" \
  --dead-letter-topic="$DLQ" \
  --max-delivery-attempts=10 \
  --min-retry-delay=10s \
  --max-retry-delay=600s \
  --ack-deadline=30 \
  --project "$PROJECT_ID"
```

## Cloud Run Deployment (Recommended)
Cloud Run is recommended for this worker because it is operationally simple, supports service account auth natively, and scales by demand without idle VM/GKE overhead.

```bash
REGION="us-central1"
WORKER_SA="analytics-worker-sa"

gcloud iam service-accounts create "$WORKER_SA" --project "$PROJECT_ID"

gcloud projects add-iam-policy-binding "$PROJECT_ID" \
  --member="serviceAccount:${WORKER_SA}@${PROJECT_ID}.iam.gserviceaccount.com" \
  --role="roles/pubsub.subscriber"

gcloud builds submit --tag "gcr.io/$PROJECT_ID/analytics-worker:latest" analytics-worker

gcloud run deploy analytics-worker \
  --image "gcr.io/$PROJECT_ID/analytics-worker:latest" \
  --region "$REGION" \
  --service-account "${WORKER_SA}@${PROJECT_ID}.iam.gserviceaccount.com" \
  --set-env-vars "GOOGLE_CLOUD_PROJECT=$PROJECT_ID,ANALYTICS_PUBSUB_TOPIC=$TOPIC,ANALYTICS_PUBSUB_SUBSCRIPTION=$SUB,ANALYTICS_PUBSUB_DEAD_LETTER_TOPIC=$DLQ,ANALYTICS_MONGO_URI=__SET__,ANALYTICS_DB_NAME=analytics" \
  --memory 1Gi --cpu 1 --max-instances 25 --min-instances 0 \
  --no-allow-unauthenticated
```

## Performance Design
- Tracking endpoint is async/non-blocking and returns `202` quickly
- Browser batches events (no per-click network writes)
- Worker uses `bulkWrite` with high-throughput batches
- Idempotency by unique `eventId` and upsert logic
- Connection pooling enabled in backend and worker
- Duplicate Pub/Sub deliveries are safely ignored
- Worker health snapshots expose queue depth + throughput

## Security and Privacy
- Sensitive metadata fields are redacted
- No password/token secrets stored in analytics payloads
- Consent endpoint: `POST /api/track/consent`
- DNT/GPC respected (`dnt: 1` / `sec-gpc: 1`)
- Optional IP anonymization via `ANALYTICS_MASK_IP=true`
- Optional raw-event retention via TTL (`ANALYTICS_RAW_EVENTS_TTL_DAYS`)
